## ğŸ“– Apa itu Async/Await?

**Async/Await** adalah cara alternatif untuk mengkonsumsi Promise. Ini adalah syntactic sugar yang membuat asynchronous code terlihat dan berjalan seperti synchronous code.

### ğŸ”‘ Konsep Utama:

- **`async`** = Keyword yang membuat function menjadi asynchronous
- **`await`** = Keyword yang menunggu Promise selesai sebelum melanjutkan ke baris berikutnya

### ğŸ’¡ Analogi Sederhana:
Bayangkan kamu memesan makanan di restoran:
- **Tanpa await**: Kamu pesan, langsung pergi tanpa tunggu makanan (asynchronous)
- **Dengan await**: Kamu pesan, **tunggu sampai makanan siap**, baru kamu ambil (terlihat synchronous)

## âš–ï¸ Langkah 5: Perbandingan .then() vs Async/Await

Mari bandingkan kedua cara untuk **tugas yang sama**.

### ğŸ”´ Cara Lama dengan `.then()`:

```javascript

      // Semua yang pakai data HARUS di sini
    });
```

### ğŸŸ¢ Cara Baru dengan Async/Await:

```javascript

  const res = await fetch('https://jsonplaceholder.typicode.com/users');
  const data = await res.json();

  // Data bisa dipakai di sini, dan di bawahnya
}
```

### ğŸ“Š Perbandingan Visual:

| Aspek | .then() | Async/Await |
|-------|---------|-------------|
| **Readability** | â­â­â­ | â­â­â­â­â­ |
| **Struktur** | Nested callbacks | Linear (top to bottom) |
| **Scope data** | Di dalam callback | Di function scope |
| **Debugging** | Susah | Lebih mudah |
| **Code length** | Lebih panjang | Lebih pendek |

### ğŸ’¡ Insight:

Lihat perbedaannya:

**`.then()` version:**
- Data `data` hanya bisa dipakai **di dalam callback kedua**
- Kalau mau pakai data di tempat lain, harus di dalam `.then()`

**`Async/Await` version:**
- Data `data` bisa dipakai **di mana saja dalam function**
- Lebih fleksibel dan clean
