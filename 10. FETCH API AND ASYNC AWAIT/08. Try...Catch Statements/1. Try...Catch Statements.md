## ğŸ¯ Pengenalan

Dalam video tutorial ini, kita akan belajar tentang **try-catch statement** di JavaScript. Ini adalah cara yang sangat penting untuk menangani error (kesalahan) yang mungkin terjadi dalam kode kita.

**Mengapa ini penting?**
- Try-catch memungkinkan kita menulis kode khusus jika terjadi error
- Try-catch juga memungkinkan kita menulis kode khusus jika **tidak** ada error
- Ini adalah cara paling umum untuk menangani error saat menggunakan **async/await**

## ğŸ“– Apa itu Try-Catch?

**Try-Catch** adalah sebuah sintaks di JavaScript yang memungkinkan kita untuk:
- âœ… **"Mencoba" (try)** menjalankan kode yang mungkin menghasilkan error
- âš ï¸ **"Menangkap" (catch)** error tersebut jika terjadi
- ğŸ›¡ï¸ Mencegah program kita berhenti total karena error

**Analogi sederhana:**
Bayangkan kamu mencoba (try) melempar bola ke keranjang. Jika meleset, kamu siap menangkap (catch) bola itu agar tidak hilang.

## ğŸ¤” Mengapa Perlu Try-Catch?

### âŒ Tanpa Try-Catch

Ketika kita menulis kode seperti ini:


Di browser console, kita akan melihat:

### âœ… Dengan Try-Catch

<a name="sintaks-dasar-try-catch"></a>
## ğŸ”§ Sintaks Dasar Try-Catch

Berikut adalah struktur dasar try-catch:

```javascript
try {
  // Kode yang mungkin menghasilkan error
  // Letakkan di sini
} catch (error) {
  // Kode yang akan dijalankan jika ada error
  // error = informasi tentang error yang terjadi
}
```

**Penjelasan:**
- ğŸ”¹ **`try { }`** - Blok kode yang akan "dicoba" dijalankan
- ğŸ”¹ **`catch (error) { }`** - Blok kode yang akan dijalankan **hanya jika** terjadi error
- ğŸ”¹ **`error`** - Variabel yang menyimpan informasi tentang error (bisa diberi nama `e`, `err`, `error`, terserah kamu)

## â° Kapan Menggunakan Try-Catch?

### âš ï¸ Jangan Berlebihan!

Kamu **tidak perlu** menggunakan try-catch di setiap baris kode. Gunakan try-catch hanya ketika:

1. ğŸŒ **Membuat HTTP Request / API Call**

2. ğŸ“‚ **Membaca File**
   - File mungkin tidak ada atau rusak

3. ğŸ”„ **Operasi yang Tidak Pasti Hasilnya**

   - Kamu punya rencana untuk menangani kegagalan tersebut

### ğŸ’¡ Contoh Penggunaan yang Baik

## ğŸ¨ Membuat Error Sendiri

Kita tidak hanya bisa menangkap error yang terjadi otomatis, tapi juga bisa **membuat dan melempar (throw) error sendiri**!

### ğŸ“ Contoh: Fungsi Double dengan Validasi

Mari kita buat fungsi yang menggandakan angka, dengan validasi:

```javascript

  if (isNaN(number)) {
    // Jika bukan angka, lempar error
    throw new Error(number + ' is not a number');
  }
  
  // Jika angka, kembalikan nilai yang digandakan
  return number * 2;
}
```

**Penjelasan Kode:**
- ğŸ”¹ **`isNaN(number)`** - Fungsi bawaan JavaScript untuk mengecek apakah sesuatu **bukan angka** (is Not a Number)
- ğŸ”¹ **`throw new Error(...)`** - Melempar/membuat error baru dengan pesan custom
- ğŸ”¹ Jika input valid (adalah angka), fungsi akan return `number * 2`

### ğŸ§ª Testing Fungsi Double

#### âœ… Test dengan Input yang Benar (Angka)

```javascript

  const y = double(1);
  console.log(y); // Output: 2
} catch (error) {
  console.log(error);
}
```

**Output:**
```
2
```

âœ… Tidak ada error, fungsi berjalan normal!

#### âŒ Test dengan Input yang Salah (String)

```javascript

  console.log(y);
} catch (error) {
  console.log(error); // Error tertangkap di sini!
}
```

**Output:**
```

```

**Apa yang terjadi?**
1. â¡ï¸ Kode dalam `try` dijalankan
2. â¡ï¸ Fungsi `double('hello')` dipanggil
3. â¡ï¸ `isNaN('hello')` bernilai `true` (karena 'hello' bukan angka)
4. â¡ï¸ `throw new Error(...)` dieksekusi
5. â¡ï¸ Langsung loncat ke blok `catch`
6. â¡ï¸ Error ditampilkan di console
