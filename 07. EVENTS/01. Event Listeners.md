<a name="metode-3-addeventlistener"></a>
### âœ… Metode 3: addEventListener() (Disarankan!)

Ini adalah cara **modern dan paling direkomendasikan** untuk menambahkan event listener.

#### ğŸŒŸ Keuntungan:

- âœ… Bisa menambahkan **multiple event handler** pada event yang sama
- âœ… Lebih fleksibel dan mudah di-maintain
- âœ… Bisa menghapus event listener dengan `removeEventListener()`
- âœ… Best practice di industri

#### Syntax:

```javascript
element.addEventListener(eventType, callbackFunction);
```

**Parameter:**
- `eventType`: String nama event (misalnya: `'click'`, `'submit'`)
- `callbackFunction`: Fungsi yang akan dijalankan ketika event terjadi

#### Contoh Penggunaan:

**1. Dengan Anonymous Function:**

```javascript
const clearBtn = document.querySelector('#clear');

clearBtn.addEventListener('click', function() {
  alert('Clear Items');
});
```

**2. Dengan Arrow Function (Lebih Modern):**

```javascript
clearBtn.addEventListener('click', () => {
  alert('Clear Items');
});
```

**3. Arrow Function One-Liner:**

```javascript
clearBtn.addEventListener('click', () => alert('Clear Items'));
```

**4. Dengan Named Function:**

```javascript
function onClear() {
  alert('Clear Items');
}

// âœ… Tanpa parentheses - passing reference
clearBtn.addEventListener('click', onClear);

// âŒ Dengan parentheses - langsung execute!
clearBtn.addEventListener('click', onClear()); // SALAH!
```

> ğŸ’¡ **Penting**: Saat menggunakan named function, **jangan tambahkan ()** setelah nama fungsi. Kita ingin pass **reference** ke fungsi, bukan **eksekusi** fungsinya.

#### Multiple Event Handlers:

Keuntungan besar `addEventListener()` adalah bisa menambahkan **beberapa handler** untuk event yang sama:

```javascript
// Event handler pertama
clearBtn.addEventListener('click', () => {
  alert('Clear Items');
});

// Event handler kedua - KEDUANYA AKAN JALAN!
clearBtn.addEventListener('click', () => {
  console.log('Clear Items');
});
```

Ketika button diklik:
1. âœ… Alert akan muncul
2. âœ… Setelah alert di-close, console.log akan jalan

> ğŸ“ **Note**: Alert adalah **blocking operation** (synchronous), jadi `console.log` akan menunggu alert selesai.

<a name="menghapus-event-listener"></a>
## ğŸ—‘ï¸ Menghapus Event Listener

Untuk menghapus event listener, gunakan `removeEventListener()`.

### Kapan Perlu Menghapus Event Listener?

- ğŸ¯ Ketika elemen akan dihapus dari DOM
- ğŸ¯ Untuk menghemat memory (mencegah memory leak)
- ğŸ¯ Ketika event listener tidak diperlukan lagi

### Syntax:

```javascript
element.removeEventListener(eventType, functionReference);
```

> âš ï¸ **Penting**: Untuk menghapus event listener, kita **harus menggunakan named function** (bukan anonymous function).

### Contoh:

```javascript
const clearBtn = document.querySelector('#clear');

function onClear() {
  alert('Clear Items');
}

// Menambahkan event listener
clearBtn.addEventListener('click', onClear);

// Menghapus event listener setelah 5 detik
setTimeout(() => {
  clearBtn.removeEventListener('click', onClear);
  console.log('Event listener removed!');
}, 5000);
```

**Penjelasan:**
- `setTimeout()` akan menjalankan fungsi setelah delay tertentu (dalam milliseconds)
- `5000` milliseconds = 5 detik
- Setelah 5 detik, event listener akan dihapus
- Button tidak lagi merespon klik setelah 5 detik

> ğŸ’¡ **Tips**: `setTimeout()` adalah fungsi **asynchronous** yang akan kita pelajari lebih dalam nanti.

<a name="cara-2-foreach-remove"></a>
### ğŸ”„ Cara 2: Menggunakan forEach() dan remove()

**Lebih clean dan modern.**

```javascript

```

**Cara Kerja:**
1. `querySelectorAll('li')` mengambil **semua** elemen `<li>` dalam bentuk NodeList
2. `forEach()` loop melalui setiap item
3. `remove()` menghapus setiap item dari DOM

**Pro & Cons:**
- âœ… **Pro**: Lebih clean dan readable
- âœ… **Pro**: Modern ES6+ syntax
- âš ï¸ **Cons**: Sedikit lebih lambat dari while loop
- âœ… **Pro**: Cleanup lebih baik (event listeners dibersihkan dengan proper)

<a name="cara-3-while-loop"></a>
### âš¡ Cara 3: Menggunakan While Loop (Paling Performant!)

**Cara paling efficient untuk menghapus multiple elements.**

```javascript
function onClear() {
  const itemList = document.querySelector('#item-list');
  
  while (itemList.firstChild) {
    itemList.removeChild(itemList.firstChild);
  }
}

clearBtn.addEventListener('click', onClear);
```

**Cara Kerja:**
1. `while (itemList.firstChild)` - Cek apakah ada child pertama
2. Selama ada child, loop akan terus berjalan
3. `removeChild(itemList.firstChild)` - Hapus child pertama
4. Loop berhenti ketika tidak ada child lagi

**Kenapa Ini Paling Performant?**
- Loop langsung menghapus child pertama berulang kali
- Tidak perlu create array atau NodeList terlebih dahulu
- Memory efficient

**Pro & Cons:**
- âœ… **Pro**: Paling performant
- âœ… **Pro**: Memory efficient
- âœ… **Pro**: Recommended untuk large lists
- âš ï¸ **Cons**: Syntax agak kurang readable untuk pemula

## ğŸ“ Kesimpulan

### ğŸ“Œ Poin-Poin Penting:

1. **Event** adalah aksi atau kejadian yang terjadi di website (click, hover, submit, dll)

- âœ… Cleanup event listener saat element dihapus dari DOM
