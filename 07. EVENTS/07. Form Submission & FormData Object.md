## ğŸ¯ Pengenalan

Dalam pengembangan web, **form submission** adalah proses mengirimkan data dari form ke server atau memproses data tersebut di sisi client (browser). 

### ğŸ¤” Kapan Menggunakan Form Submission?

- âœ… Ketika ingin mengirim data ke backend (Node.js, PHP, dll)

### âš¡ Perbedaan Backend vs Frontend Form Processing

| Backend Processing | Frontend Processing |
|-------------------|---------------------|
| Form memiliki `action` attribute | Form tidak memiliki `action` atau `action=""` |
| Data dikirim ke file server | Data diproses di JavaScript |
| Halaman akan reload | Halaman tidak reload |

<a name="prevent-default-behavior"></a>
## ğŸ›‘ Prevent Default Behavior

Ketika form di-submit, browser secara default akan:
1. ğŸ”„ Reload halaman
2. ğŸ“¤ Mengirim data ke URL di attribute `action`

Untuk mencegah ini, kita gunakan `preventDefault()`:

```javascript
  console.log('Form submitted!');
```

### ğŸ” Apa yang Terjadi Tanpa preventDefault()?

```javascript
  // Halaman akan reload dan console log hilang
}
```

### âœ… Dengan preventDefault():

<a name="menggunakan-get-method"></a>
## ğŸ” Menggunakan `get()` Method

Method `get()` digunakan untuk mengambil nilai **satu field** berdasarkan attribute `name`.

```javascript

  // Output: "Milk" "1"

```

### âš ï¸ Penting: Attribute `name` Harus Ada!

```html
<!-- âœ… BENAR - Ada name attribute -->

<!-- âŒ SALAH - Tidak ada name attribute -->

```

Jika tidak ada `name`, `formData.get()` akan return `null`.
