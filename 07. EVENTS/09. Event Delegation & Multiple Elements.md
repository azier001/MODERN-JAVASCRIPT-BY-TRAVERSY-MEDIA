## ğŸ“– Apa itu Event Delegation?

**Event Delegation** adalah pola (pattern) dalam JavaScript di mana kita menempatkan **satu event listener pada parent element**, bukan pada setiap child element.

### ğŸ­ Analogi Sederhana:

Bayangkan kamu punya **10 anak** dan ingin mengawasi mereka:
- âŒ **Cara Tidak Efisien**: Menempatkan 1 pengawas untuk setiap anak (10 pengawas)
- âœ… **Cara Efisien**: Menempatkan 1 pengawas untuk mengawasi semua anak (1 pengawas)

Event Delegation bekerja seperti cara efisien di atas! ğŸš€

<a name="etarget-vs-ecurrenttarget"></a>
### ğŸ¯ `e.target` vs `e.currentTarget`

Ini adalah konsep yang **SANGAT PENTING** untuk dipahami!

#### `e.target` ğŸ¯
- Element yang **BENAR-BENAR DI-CLICK**
- Bisa element mana saja di dalam parent

```javascript
list.addEventListener('click', (e) => {
  console.log(e.target); 
  // Output: <li> jika kamu click item
  // Output: <ul> jika kamu click area kosong
});
```

#### `e.currentTarget` ğŸ“
- Element yang **PUNYA EVENT LISTENER**
- Selalu sama (dalam kasus ini: `<ul>`)

```javascript
list.addEventListener('click', (e) => {
  console.log(e.currentTarget); 
  // Output: SELALU <ul>
  // Karena event listener ada di <ul>
});
```

**ğŸ“ Catatan Penting:**
> Untuk Event Delegation, kita **SELALU** menggunakan `e.target`, bukan `e.currentTarget`!

<a name="tagname-property"></a>
### ğŸ·ï¸ `tagName` Property

`tagName` memberikan nama tag HTML dalam **HURUF BESAR**.

```javascript

console.log(li.tagName); // Output: "LI"

console.log(div.tagName); // Output: "DIV"
```

**âš ï¸ Perhatian:**
- `tagName` selalu dalam **UPPERCASE** (huruf besar)
- Jadi gunakan `'LI'`, bukan `'li'`

```javascript

// âŒ Salah
if (e.target.tagName === 'li') { } // Tidak akan bekerja!
```
