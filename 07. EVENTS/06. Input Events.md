## ğŸ“– Apa itu Input Events?

**Input Events** adalah kejadian yang terjadi ketika user berinteraksi dengan elemen form di halaman web. Beberapa event yang paling sering digunakan:

| Event | Deskripsi | Kapan Terjadi |
|-------|-----------|---------------|
| `input` | ğŸ”„ Dijalankan setiap kali nilai berubah | Setiap kali user mengetik atau mengubah nilai |
| `change` | ğŸ”€ Dijalankan ketika nilai berubah dan user keluar dari input | Saat user selesai mengubah dan pindah ke elemen lain |
| `focus` | ğŸ¯ Dijalankan ketika input mendapat fokus | Saat user klik ke dalam input |
| `blur` | ğŸ‘‹ Dijalankan ketika input kehilangan fokus | Saat user klik keluar dari input |

> **ğŸ’¡ Catatan Penting**: Event `input` lebih fleksibel daripada `keydown` karena bisa digunakan untuk berbagai jenis input (text, select, dll), sedangkan `keydown` hanya untuk keyboard.

## âŒ¨ï¸ Event: Input - Text Field

### ğŸ¯ Tujuan
Menangkap setiap perubahan pada text input dan menampilkannya secara real-time.

### ğŸ“ Kode

```javascript

  heading.textContent = e.target.value;
}

```

**ğŸ” Penjelasan Step by Step:**

1. **`function onInput(e)`** â†’ Membuat fungsi yang akan dijalankan ketika event terjadi
   - Parameter `e` adalah **event object** yang berisi informasi tentang event

2. **`e.target`** â†’ Elemen yang memicu event (dalam hal ini adalah `itemInput`)

3. **`e.target.value`** â†’ Nilai yang ada di dalam input field

4. **`heading.textContent`** â†’ Mengubah teks di dalam `<h1>`

5. **`addEventListener('input', onInput)`** â†’ Mendengarkan event `input` dan menjalankan fungsi `onInput`

### ğŸ¬ Hasil
Ketika user mengetik di text field, heading akan berubah secara **real-time**!

```
User mengetik: "A"     â†’ Heading: "A"
User mengetik: "Ap"    â†’ Heading: "Ap"
User mengetik: "Apple" â†’ Heading: "Apple"
```

### ğŸ†š Perbedaan `input` vs `keydown`

| Event | Kapan Terjadi | Kelebihan |
|-------|---------------|-----------|
| `keydown` | Setiap kali tombol ditekan | Bisa deteksi tombol spesifik (Enter, Esc, dll) |
| `input` | Setiap kali nilai berubah | Bekerja untuk semua jenis input (text, select, paste) |

> **âš ï¸ Rekomendasi**: Gunakan `input` untuk menangkap perubahan nilai form karena lebih universal.

## ğŸ”½ Event: Change - Select List

### ğŸ¯ Tujuan

### ğŸ“ Kode

**ğŸ” Penjelasan:**

- `e.target.value` akan berisi nilai dari `value` attribute di `<option>`

### ğŸ¬ Contoh

> **ğŸ’¡ Catatan**: Yang ditangkap adalah **value attribute**, bukan teks yang terlihat oleh user!

### ğŸ†š Perbedaan `input` vs `change` untuk Select

| Event | Kapan Terjadi |
|-------|---------------|
| `input` | âœ… Segera setelah pilihan berubah |
| `change` | âœ… Setelah pilihan berubah (lebih umum digunakan) |

Untuk select list, **keduanya sama efektifnya**. Gunakan `change` karena lebih semantic (menunjukkan perubahan).

### ğŸ“ Kode

**ğŸ” Penjelasan Step by Step:**

3. **Ternary Operator** â†’ `isChecked ? 'Checked' : 'Not Checked'`

### ğŸ”‘ Poin Penting

|-------------|-------------------------|

## ğŸ¯ Event: Focus dan Blur

### ğŸ¯ Tujuan
Memberikan **feedback visual** ketika user klik masuk atau keluar dari input field.

### ğŸ“ Kode

```javascript

  itemInput.style.outlineWidth = '1px';
  itemInput.style.outlineColor = 'red';

  console.log('Input is not focused');
  itemInput.style.outlineStyle = 'none';

itemInput.addEventListener('blur', onBlur);
```

**ğŸ” Penjelasan Step by Step:**

### ğŸ”µ Event `focus`

1. **Kapan terjadi?** â†’ Ketika user klik **ke dalam** input field
2. **Apa yang dilakukan?**
   - Menambahkan outline merah solid dengan lebar 1px
   - `itemInput.style.outlineStyle = 'solid'` â†’ Jenis outline
   - `itemInput.style.outlineWidth = '1px'` â†’ Ketebalan outline
   - `itemInput.style.outlineColor = 'red'` â†’ Warna outline

### ğŸ”´ Event `blur`

1. **Kapan terjadi?** â†’ Ketika user klik **keluar** dari input field
2. **Apa yang dilakukan?**
   - Menghilangkan outline dengan set ke `'none'`

### ğŸ¬ Visual Flow

```
User klik ke dalam input â†’ [Input dengan border merah]
User klik keluar         â†’ [Input normal tanpa border]
```

### ğŸ¨ Kenapa Pakai Focus & Blur?

Event ini sangat berguna untuk:

âœ… **User Experience (UX)** â†’ Memberikan feedback visual yang jelas  
âœ… **Accessibility** â†’ Membantu user tahu input mana yang sedang aktif  
âœ… **Validasi Form** â†’ Menampilkan error message ketika user keluar dari input  
âœ… **Styling Dinamis** â†’ Mengubah tampilan form secara interaktif

### ğŸ’¡ Contoh Penggunaan Lain

```javascript

  itemInput.style.backgroundColor = '#fffacd'; // Highlight kuning
}

function onBlur() {
  itemInput.style.backgroundColor = 'white'; // Kembali normal
}
```

<a name="tips-untuk-pemula"></a>
## ğŸ’¡ Tips untuk Pemula

### âœ… Do's (Yang Harus Dilakukan)

1. **Gunakan `input` event** untuk text field dan textarea
2. **Gunakan `change` event** untuk select list
3. **Gunakan `e.target.checked`** untuk checkbox, bukan `value`
4. **Gunakan `focus` dan `blur`** untuk feedback visual
5. **Selalu console.log()** untuk debugging saat belajar

### âŒ Don'ts (Yang Harus Dihindari)

1. âŒ Jangan gunakan `keydown` untuk menangkap nilai input (gunakan `input`)
2. âŒ Jangan lupa parameter `e` di function event handler
3. âŒ Jangan gunakan `e.target.value` untuk checkbox (gunakan `checked`)
4. âŒ Jangan langsung ubah DOM tanpa `addEventListener()`

### ğŸ”¥ Best Practices

```javascript

  console.log(e.target.value);
}

// âŒ BAD: Inline function yang sulit di-debug

```
