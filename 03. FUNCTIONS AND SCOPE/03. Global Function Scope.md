<a name="apa-itu-scope"></a>
## ğŸ“– Apa itu Scope?

**Scope** adalah **area atau konteks** di mana sebuah variabel atau kode dapat diakses.

> ğŸ”‘ **Aturan Penting:** Ada aturan khusus tentang apa yang bisa kita akses di scope tertentu!

### ğŸ¨ Jenis-jenis Scope:

<a name="variable-shadowing"></a>
## ğŸ­ Variable Shadowing

**Variable Shadowing** terjadi ketika kita membuat variabel dengan **nama yang sama** di scope yang berbeda.

```javascript
const x = 100; // Global scope

function add() {
  const x = 1; // Function scope - MENIMPA x global!
  const y = 50;
  console.log(x + y); // Pakai x yang local (1), bukan global (100)
}

add(); // Output: 51 (bukan 150!)
```

> ğŸ” **Apa yang Terjadi?**
> - Di dalam function `add`, ada variabel `x` yang baru (nilai 1)
> - Variabel lokal ini "menutupi" variabel global `x`
> - Jadi yang dipakai adalah `x` lokal (1) + `y` (50) = 51

**âš–ï¸ Prioritas Akses Variabel:**
1. **Local scope** (di dalam function saat ini) - **Prioritas Tertinggi** ğŸ¥‡
2. **Parent scope** (scope di luarnya)
3. **Global scope** (paling luar)

```javascript
const x = 100; // Global

function outer() {
  const x = 50; // Outer function scope
  
  function inner() {
    const x = 1; // Inner function scope
    console.log(x); // Pakai yang paling dekat: 1
  }
  
  inner();
}

outer(); // Output: 1
```

<a name="local-scope"></a>
## ğŸ“ Local Scope

**Local Scope** adalah istilah umum yang merujuk pada **scope tempat kita berada saat ini**.

> ğŸ’¡ **Analogi Sederhana:** "Local" = "Di sini, di tempat ini"

**Contoh:**

```javascript

  const y = 50; // Di sini, local scope = function add scope
  console.log(y); // Akses variabel local
}
```

**ğŸ¯ Kesimpulan Local Scope:**
- Jika kita bicara tentang variabel di **dalam function**, local scope-nya adalah **function tersebut**
- Jika kita bicara tentang variabel di **global**, local scope-nya adalah **global scope**
- "Local" selalu relatif terhadap **di mana kita berada**
