Dalam JavaScript, ada **dua cara utama** untuk membuat function:
1. **Function Declaration** - Cara tradisional dengan keyword `function` di awal

## ğŸ“˜ Function Declaration

<a name="apa-itu-function-declaration"></a>
### ğŸ’¡ Apa itu Function Declaration?

**Function Declaration** adalah cara membuat function dengan langsung menggunakan keyword `function` diikuti nama function-nya.

**Ciri-ciri:**
- âœ… Dimulai dengan keyword `function`
- âœ… Harus memiliki nama
- âœ… Tidak menggunakan semicolon (`;`) di akhir
- âœ… Bisa dipanggil sebelum didefinisikan (karena hoisting)

<a name="contoh-kode-function-declaration"></a>
### ğŸ“ Contoh Kode Function Declaration


**Penjelasan Kode:**
- `function` â†’ Keyword untuk membuat function
- `addDollarSign` â†’ Nama function yang kita buat
- `(value)` â†’ Parameter yang akan diterima function
- `return '$' + value` â†’ Mengembalikan string dengan simbol `$` di depan value

## ğŸ“— Function Expression

<a name="apa-itu-function-expression"></a>
### ğŸ’¡ Apa itu Function Expression?

**Function Expression** adalah function yang disimpan di dalam sebuah variable. Function ini diperlakukan seperti value biasa.

**Ciri-ciri:**
- âœ… Disimpan dalam variable (`const`, `let`, atau `var`)
- âœ… Function bisa tanpa nama (anonymous function)
- âœ… Menggunakan semicolon (`;`) di akhir
- âœ… Tidak bisa dipanggil sebelum didefinisikan

<a name="contoh-kode-function-expression"></a>
### ğŸ“ Contoh Kode Function Expression

```javascript

  return '+' + value;

```

**Penjelasan Kode:**
- `const addPlusSign` â†’ Variable yang menyimpan function
- `= function(value)` â†’ Function tanpa nama (anonymous)
- `return '+' + value` â†’ Mengembalikan string dengan simbol `+` di depan value
- `;` â†’ Semicolon di akhir (karena ini adalah statement variable)

<a name="cara-memanggil-function-expression"></a>
### ğŸ® Cara Memanggil Function Expression

**Perhatian:** Kamu TIDAK bisa memanggil function expression sebelum deklarasinya!

```javascript

  return '+' + value;

```

**Error yang muncul:**

<a name="hoisting"></a>
## ğŸˆ Hoisting

<a name="apa-itu-hoisting"></a>
### ğŸ¤” Apa itu Hoisting?

**Hoisting** adalah proses dimana JavaScript **"mengangkat"** (hoist) semua deklarasi function dan variable ke bagian **paling atas** dari scope sebelum kode dijalankan.

**Analogi sederhana:**
Bayangkan JavaScript membaca kode kamu **dua kali**:
1. **Scan pertama:** Mencari semua deklarasi function dan variable
2. **Scan kedua:** Menjalankan kode secara normal

Ini seperti JavaScript membuat "daftar" semua function yang ada sebelum mulai menjalankan program! ğŸ“‹

<a name="hoisting-pada-function-declaration"></a>
### âœ… Hoisting pada Function Declaration

Function Declaration **di-hoist sepenuhnya**, jadi bisa dipanggil sebelum deklarasinya!

```javascript
// Ini BERHASIL karena hoisting âœ…
console.log(addDollarSign(100)); // Output: $100

function addDollarSign(value) {
  return '$' + value;
}
```

**Yang terjadi di balik layar:**
```javascript
// JavaScript secara internal melakukan ini:

// 1. Hoisting - function "diangkat" ke atas
function addDollarSign(value) {
  return '$' + value;
}

// 2. Lalu kode kamu dijalankan
console.log(addDollarSign(100));
```

<a name="hoisting-pada-function-expression"></a>
### âŒ Hoisting pada Function Expression

Function Expression **hanya variable-nya** yang di-hoist, bukan function-nya!

```javascript
// Ini GAGAL âŒ
console.log(addPlusSign(200)); 
// Error: Cannot access 'addPlusSign' before initialization

const addPlusSign = function (value) {
  return '+' + value;
};
```

**Yang terjadi di balik layar:**
```javascript
// JavaScript secara internal melakukan ini:

// 1. Hoisting - hanya deklarasi variable
let addPlusSign; // undefined, belum ada function-nya

// 2. Kode kamu dijalankan
console.log(addPlusSign(200)); // Error! Masih undefined

// 3. Baru assignment function terjadi
addPlusSign = function (value) {
  return '+' + value;
};
```

<a name="kesimpulan"></a>
## ğŸ“ Kesimpulan

### ğŸ“Œ Kapan Menggunakan Function Declaration?

**Gunakan Function Declaration jika:**
- âœ… Kamu ingin function bisa dipanggil dari mana saja (karena hoisting)
- âœ… Kamu ingin kode lebih tradisional dan mudah dibaca
- âœ… Kamu suka menempatkan semua function di bagian bawah file

### ğŸ“Œ Kapan Menggunakan Function Expression?

**Gunakan Function Expression jika:**
- âœ… Kamu ingin kontrol yang lebih ketat (harus deklarasi dulu baru pakai)
- âœ… Kamu akan menggunakan function sebagai callback
- âœ… Kamu ingin mengikuti style modern JavaScript (dengan `const`)

   - Kode jadi lebih mudah dibaca dan dipahami
