<a name="pengenalan"></a>
## 🎯 Pengenalan

Sebelumnya kita sudah belajar soal **property flags** — yaitu pengaturan `configurable`, `enumerable`, dan `writable` untuk tiap properti objek secara satu per satu.

Nah, sekarang kita akan belajar cara yang lebih praktis: **seal** dan **freeze**. Dua metode ini memungkinkan kita mengunci seluruh properti objek sekaligus, tanpa harus mengatur satu per satu.

Perbedaan singkatnya:

| Operasi | Tambah properti | Hapus properti | Ubah nilai |
|---|---|---|---|
| `Object.seal()` | ❌ Tidak bisa | ❌ Tidak bisa | ✅ Bisa |
| `Object.freeze()` | ❌ Tidak bisa | ❌ Tidak bisa | ❌ Tidak bisa |

<a name="object-seal"></a>
## 🔒 Object.seal() — Mengunci Struktur

`Object.seal()` mencegah properti **ditambah atau dihapus** dari sebuah objek. Tapi kamu masih **boleh mengubah nilai** properti yang sudah ada.

### Contoh Objek

Kita mulai dengan objek `rectObj` yang punya tiga properti:

```javascript
const rectObj = {
  name: 'Rectangle 1',
  width: 10,
  height: 10,
};
```

### Menyegel Objek

Gunakan `Object.seal()` dan lempar objeknya ke dalam:

```javascript
Object.seal(rectObj);
```

### Mengecek Flags Setelah Diseal

Setelah diseal, kita bisa lihat semua flag-nya berubah. Gunakan `getOwnPropertyDescriptors()` untuk cek semua properti sekaligus:

```javascript
let descriptors = Object.getOwnPropertyDescriptors(rectObj);
console.log(descriptors);
```

> 💡 Yang berubah setelah `seal`: flag **`configurable` menjadi `false`** untuk semua properti. Artinya, properti tidak bisa ditambah atau dihapus.

### Apa yang Tidak Bisa Dilakukan?

```javascript
// ❌ Menambah properti baru — tidak akan berhasil
rectObj.color = 'red';

// ❌ Menghapus properti — tidak akan berhasil
delete rectObj.name;

console.log(rectObj);
// Output: { name: 'Rectangle 1', width: 10, height: 10 }
// 'color' tidak muncul, 'name' tetap ada
```

### Apa yang Masih Bisa Dilakukan?

```javascript
// ✅ Mengubah nilai properti yang sudah ada — masih boleh!
rectObj.width = 20;

console.log(rectObj);
// Output: { name: 'Rectangle 1', width: 20, height: 10 }
```

<a name="object-freeze"></a>
## ❄️ Object.freeze() — Mengunci Segalanya

`Object.freeze()` jauh lebih ketat dari seal. Setelah difreeze, objek **benar-benar tidak bisa diapa-apain** — tidak bisa ditambah, dihapus, maupun diubah.

### Contoh Objek

```javascript
const circleObj = {
  name: 'Circle 1',
  radius: 30,
};
```

### Membekukan Objek

```javascript
Object.freeze(circleObj);
```

### Mengecek Flags Setelah Difreeze

```javascript
let descriptors = Object.getOwnPropertyDescriptors(circleObj);
console.log(descriptors);
```

> 💡 Yang berubah setelah `freeze`: **`configurable` DAN `writable` keduanya menjadi `false`**. Ini yang membedakannya dari seal — `writable: false` itulah yang mencegah perubahan nilai.

### Semua Operasi Diblokir

```javascript
// ❌ Menambah properti baru — tidak berhasil
circleObj.color = 'blue';

// ❌ Menghapus properti — tidak berhasil
delete circleObj.name;

// ❌ Mengubah nilai properti — tidak berhasil!
circleObj.name = 'New Name';

console.log(circleObj);
// Output: { name: 'Circle 1', radius: 30 }
// Semuanya tetap seperti semula
```

<a name="cek-status"></a>
## 🔬 Cek Status: isSealed & isFrozen

JavaScript punya dua metode untuk mengecek apakah sebuah objek sudah diseal atau difreeze:

```javascript
// Cek apakah rectObj sudah diseal
console.log('rectObj is sealed?', Object.isSealed(rectObj));
// Output: true ✅

// Cek apakah rectObj sudah difreeze
console.log('rectObj is frozen?', Object.isFrozen(rectObj));
// Output: false ❌ (kita cuma seal, tidak freeze)

// Cek apakah circleObj sudah diseal
console.log('circleObj is sealed?', Object.isSealed(circleObj));
// Output: true ✅

// Cek apakah circleObj sudah difreeze
console.log('circleObj is frozen?', Object.isFrozen(circleObj));
// Output: true ✅
```

### 🤔 Kenapa `circleObj` bisa `isSealed = true` padahal kita cuma freeze?

Ini menarik! Ternyata **freeze otomatis juga menyegel objek**. Logikanya begini:

- `isSealed` = true ketika `configurable: false`
- `isFrozen` = true ketika `configurable: false` **DAN** `writable: false`

Karena `Object.freeze()` men-set **keduanya** ke false, maka syarat untuk `isSealed` juga terpenuhi secara otomatis.

Intinya: **semua objek yang difreeze, otomatis juga terseal — tapi tidak sebaliknya.**

<a name="rangkuman"></a>
## 💡 Rangkuman Perbandingan

```javascript
// SEAL: Bisa ubah nilai, tapi tidak bisa tambah/hapus properti
Object.seal(rectObj);

// FREEZE: Tidak bisa tambah, hapus, maupun ubah properti sama sekali
Object.freeze(circleObj);
```

| Metode | `configurable` | `writable` | Tambah | Hapus | Ubah |
|---|---|---|---|---|---|
| `Object.seal()` | `false` | tetap `true` | ❌ | ❌ | ✅ |
| `Object.freeze()` | `false` | `false` | ❌ | ❌ | ❌ |

> ⚠️ **Catatan penting:** Semua operasi yang diblokir oleh seal/freeze akan **diam-diam gagal** (tidak throw error) di mode normal. Kalau mau dapat error-nya, gunakan `"use strict"` mode.
