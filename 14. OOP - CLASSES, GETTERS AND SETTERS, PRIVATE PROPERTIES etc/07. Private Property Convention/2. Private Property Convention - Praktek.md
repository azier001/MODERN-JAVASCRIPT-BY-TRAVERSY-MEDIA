<a name="apa-itu-encapsulation"></a>
## ЁЯдФ Apa itu Encapsulation?

Encapsulation adalah konsep menyembunyikan data atau properti/method tertentu dari sebuah class. Di bahasa OOP lain seperti Java, ada keyword `private` untuk mencegah akses dari luar class. Di JavaScript, kita **tidak punya keyword itu**, tapi ada konvensi yang dipakai developer.

> ЁЯТм Intinya: kita ingin memisahkan mana yang "boleh disentuh" dari luar, dan mana yang hanya boleh dipakai di dalam class sendiri.

---

<a name="masalah-properti-yang-terlalu-terbuka"></a>
## тЪая╕П Masalah: Properti yang Terlalu Terbuka

Kita mulai dengan membuat class `Wallet` sederhana:

```javascript
class Wallet {
  constructor() {
    this.balance = 0; // properti balance, awalnya 0
  }

  deposit(amount) {
    this.balance += amount;
  }

  withdraw(amount) {
    this.balance -= amount;
  }
}

const wallet = new Wallet();
wallet.deposit(300);
wallet.withdraw(250);
console.log(wallet.balance); // 50
```

ЁЯТб **Apa yang terjadi di sini?**
- `constructor()` dijalankan otomatis saat kita buat object baru dengan `new Wallet()`
- `deposit()` dan `withdraw()` adalah method untuk mengubah balance
- Kode ini **bekerja**, tapi ada masalah...

**Masalahnya:** siapapun bisa langsung mengubah `wallet.balance` dari luar class!

```javascript
wallet.balance = 999999; // тЭМ ini bisa dilakukan! Tidak aman.
```

Kita mau agar `balance` hanya bisa diubah lewat method `deposit` dan `withdraw`.

<a name="konvensi-underscore"></a>
## ЁЯФС Konvensi Underscore `_`

Karena JavaScript tidak punya `private` keyword (untuk saat ini), developer menggunakan **konvensi underscore** `_` di depan nama properti/method untuk menandai bahwa itu **private**.

```javascript
class Wallet {
  constructor() {
    this._balance = 0; // тЖР tambahkan underscore
    this._transactions = [];
  }

  deposit(amount) {
    this._balance += amount; // тЖР pakai _balance di sini juga
  }

  withdraw(amount) {
    this._balance -= amount; // тЖР dan di sini
  }
}
```

ЁЯТб **Apa yang terjadi di sini?**
- `_balance` (dengan underscore) memberi sinyal ke developer lain: *"jangan akses properti ini langsung dari luar class"*
- Tapi ingat тАФ ini **hanya konvensi**! Secara teknis, `wallet._balance` masih bisa diakses. Underscore cuma jadi tanda peringatan, bukan penghalang nyata.

<a name="getter-cara-aman-mengakses-data-private"></a>
## ЁЯСБя╕П Getter: Cara Aman Mengakses Data Private

Meski `_balance` private, kita tetap butuh cara untuk **membaca** balance. Solusinya: gunakan **getter**.

**Langkah 1 тАФ Opsi sederhana: buat method biasa**

```javascript
class Wallet {
  constructor() {
    this._balance = 0;
  }

  getBalance() {
    return this._balance;
  }
}

const wallet = new Wallet();
console.log(wallet.getBalance()); // dipanggil seperti fungsi
```

**Langkah 2 тАФ Lebih baik: pakai `get` keyword (getter)**

```javascript
class Wallet {
  constructor() {
    this._balance = 0;
  }

  // tambahkan keyword "get" di depan nama method
  get balance() {
    return this._balance;
  }
}

const wallet = new Wallet();
console.log(wallet.balance); // dipanggil seperti properti, bukan fungsi!
```

ЁЯТб **Apa yang terjadi di sini?**
- Keyword `get` mengubah method menjadi **getter**
- Getter diakses **tanpa tanda kurung** `()` тАФ seperti mengakses properti biasa
- Di balik layar, `wallet.balance` sebenarnya memanggil fungsi yang mengembalikan `this._balance`

**Langkah 3 тАФ Tambahkan validasi di `withdraw`**

```javascript
class Wallet {
  constructor() {
    this._balance = 0;
  }

  deposit(amount) {
    this._balance += amount;
  }

  withdraw(amount) {
    // cek dulu: jangan sampai saldo minus!
    if (amount > this._balance) {
      console.log('Not enough funds');
      return; // тЖР hentikan fungsi, jangan lanjut
    }
    this._balance -= amount;
  }

  get balance() {
    return this._balance;
  }
}
```

ЁЯТб **Apa yang terjadi di sini?**
- `if (amount > this._balance)` mengecek apakah uang yang mau ditarik melebihi saldo
- Kalau iya, tampilkan pesan dan `return` тАФ fungsi berhenti, saldo tidak berubah

<a name="menambah-private-property-transactions"></a>
## ЁЯУЛ Menambah Private Property: Transactions

Sekarang kita tambahkan fitur: **mencatat semua transaksi** ke dalam array private.

```javascript
class Wallet {
  constructor() {
    this._balance = 0;
    this._transactions = []; // тЖР array untuk menyimpan riwayat transaksi
  }

  // getter untuk transactions
  get transactions() {
    return this._transactions; // тЖР kembalikan array transaksi
  }

  get balance() {
    return this._balance;
  }
}
```

ЁЯТб **Apa yang terjadi di sini?**
- `this._transactions = []` membuat array kosong saat wallet dibuat
- Getter `transactions` memberi akses **baca saja** ke array itu dari luar class
- Kamu juga bisa memproses atau memformat data dulu di sini sebelum dikembalikan, karena ini sebuah method

<a name="private-methods"></a>
## ЁЯФз Private Methods: `_processDeposit` & `_processWithdraw`

Kita pisahkan logika "mencatat transaksi" ke dalam **private method** tersendiri. Kenapa? Karena pengguna class tidak perlu tahu cara kerja pencatatan ini тАФ mereka cukup panggil `deposit()` atau `withdraw()`.

```javascript
class Wallet {
  constructor() {
    this._balance = 0;
    this._transactions = [];
  }

  // public method тАФ boleh diakses dari luar
  deposit(amount) {
    this._processDeposit(amount); // тЖР panggil private method
    this._balance += amount;
  }

  // public method тАФ boleh diakses dari luar
  withdraw(amount) {
    if (amount > this._balance) {
      console.log('Not enough funds');
      return;
    }
    this._processWithdraw(amount); // тЖР panggil private method
    this._balance -= amount;
  }

  // private method тАФ underscore menandai ini tidak untuk dipanggil dari luar
  _processDeposit(amount) {
    console.log(`Depositing ${amount}`);
    this._transactions.push({
      type: 'deposit',
      amount,
    });
  }

  // private method тАФ underscore menandai ini tidak untuk dipanggil dari luar
  _processWithdraw(amount) {
    console.log(`Withdrawing ${amount}`);
    this._transactions.push({
      type: 'withdraw',
      amount,
    });
  }

  get balance() {
    return this._balance;
  }

  get transactions() {
    return this._transactions;
  }
}
```

**Coba jalankan:**

```javascript
const wallet = new Wallet();
wallet.deposit(300);
wallet.withdraw(50);

console.log(wallet.balance);      // 250
console.log(wallet.transactions); // [{type: 'deposit', amount: 300}, {type: 'withdraw', amount: 50}]
```

ЁЯТб **Apa yang terjadi di sini?**
- `_processDeposit` dan `_processWithdraw` adalah private method тАФ dikasih prefix `_`
- Method ini **dipanggil dari dalam** `deposit()` dan `withdraw()`, bukan dari luar class
- `this._transactions.push({...})` menambahkan object baru ke array transaksi setiap kali ada deposit atau withdrawal
- Template literal `` `Depositing ${amount}` `` mencetak pesan dengan nilai amount yang dinamis

<a name="analogi-mobil--encapsulation"></a>
## ЁЯЪЧ Analogi: Mobil & Encapsulation

Encapsulation bisa dianalogikan seperti **mengendarai mobil**:

| Yang kamu **perlu tahu** (public) | Yang **tidak perlu kamu tahu** (private) |
|---|---|
| ЁЯФС Kunci kontak / ignition | тЪЩя╕П Cara kerja alternator |
| ЁЯж╢ Pedal gas & rem | ЁЯФй Mekanisme axle roda |
| ЁЯФД Setir | ЁЯТЙ Sistem injeksi bahan bakar |

Dalam kode kita:
- **Public** тЖТ `deposit()`, `withdraw()`, getter `balance`, getter `transactions`  
- **Private** тЖТ `_balance`, `_transactions`, `_processDeposit()`, `_processWithdraw()`

> ЁЯТм Kamu tidak perlu tahu cara kerja mesin untuk bisa nyetir. Begitu juga pengguna class `Wallet` тАФ mereka cukup tahu cara deposit dan withdraw, tidak perlu tahu cara data transaksi dicatat.

<a name="batasan-underscore-convention"></a>
## тЪб Batasan Underscore Convention

Perlu diingat: **underscore tidak benar-benar mencegah akses!**

```javascript
// Ini masih bisa dilakukan dan tetap bekerja ЁЯШм
console.log(wallet._balance);      // masih bisa diakses
console.log(wallet._transactions); // masih bisa diakses
```

Underscore hanya sebuah **sinyal ke developer lain** bahwa: *"tolong jangan akses ini dari luar class."* Ini soal kesepakatan dan disiplin, bukan larangan teknis.
