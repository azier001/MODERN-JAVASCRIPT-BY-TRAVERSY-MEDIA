<a name="apa-itu-getter-setter"></a>
## ğŸ” Apa itu Getter & Setter?

**Getter** dan **Setter** adalah method khusus yang dipakai untuk **mengambil** atau **mengubah** nilai properti dari sebuah object.

Yang bikin spesial: meskipun getter/setter itu sebenarnya sebuah *method* (fungsi), cara memanggilnya **seperti properti biasa** â€” tanpa kurung `()`.

---

<a name="kenapa-pakai-getter-setter"></a>
## ğŸ¤” Kenapa Pakai Getter & Setter?

Ada dua alasan utama:

1. **Mau ngelakuin sesuatu ke nilainya sebelum dikembalikan atau disimpan** â€” misalnya, memastikan huruf pertama selalu kapital.
2. **Properti private** â€” kalau kamu punya properti yang nggak mau diakses langsung dari luar, getter/setter jadi "pintu resmi"-nya.

## ğŸ—ï¸ Membuat Class Dasar

Kita mulai dengan class `Person` yang punya `firstName` dan `lastName`.

```javascript

  constructor(firstName, lastName) {
    this.firstName = firstName;
    this.lastName = lastName;
  }
}

const person1 = new Person('john', 'doe');

```

Sampai sini normal. Tapi bagaimana kalau kita selalu mau nama depan/belakang dimulai dengan huruf kapital, apapun inputnya?

<a name="masalah-naming-conflict"></a>
## âš ï¸ Masalah Naming Conflict

Kalau kita langsung bikin getter dengan nama `firstName`, akan bentrok dengan properti `this.firstName` di dalam constructor. Ini menyebabkan **rekursi tak terbatas** (getter manggil dirinya sendiri terus).

**Solusinya:** Pakai underscore `_` di depan nama properti aslinya.

```javascript
constructor(firstName, lastName) {
  this._firstName = firstName; // âœ… pakai underscore
  this._lastName = lastName;
}
```

> ğŸ’¡ Underscore `_` adalah **konvensi** yang dipakai developer untuk bilang: *"ini properti private, jangan diakses langsung dari luar."* Tapi secara teknis di JavaScript, orang masih bisa mengaksesnya â€” ini hanya kesepakatan tidak tertulis.

<a name="membuat-getter"></a>
## ğŸ“¥ Membuat Getter

Getter dibuat dengan keyword `get` sebelum nama method-nya.

```javascript
get firstName() {
  return this._firstName; // akses properti underscore
}
```

Sekarang kita bisa akses seperti ini:

```javascript
console.log(person1.firstName); // tanpa kurung ()!
```

<a name="menambahkan-logika-di-getter"></a>
## âœ¨ Menambahkan Logika di Getter

Nah, ini yang bikin getter berguna. Kita bisa tambahkan logika â€” misalnya kapitalkan huruf pertama sebelum nilai dikembalikan.

```javascript

  return this._firstName.charAt(0).toUpperCase() + this._firstName.slice(1);
}
```

Penjelasan singkat:
- `.charAt(0)` â†’ ambil karakter pertama (misal: `'j'`)
- `.toUpperCase()` â†’ jadikan kapital (`'J'`)
- `.slice(1)` â†’ ambil semua karakter *kecuali* yang pertama (`'ohn'`)
- Hasilnya digabung: `'J'` + `'ohn'` = `'John'` âœ…


<a name="membuat-setter"></a>
## ğŸ“¤ Membuat Setter

Setter dibuat dengan keyword `set`. Setter **selalu menerima satu parameter** â€” nilai yang ingin di-set.

```javascript
set firstName(value) {
  this._firstName = value.charAt(0).toUpperCase() + value.slice(1);
}
```

Sekarang kalau kita *mengubah* nama, setter ini yang akan dipanggil (bukan langsung set propertinya):

```javascript

console.log(person1.firstName); // "Joe" âœ…
```

> ğŸ”‘ Karena ada setter, saat kita nulis `person1.firstName = 'joe'`, JavaScript otomatis tahu untuk panggil fungsi setter â€” bukan langsung set propertinya.

<a name="refactor-hindari-kode-berulang"></a>
## ğŸ”§ Refactor: Hindari Kode Berulang

Kalau kita bikin getter & setter untuk `lastName` juga, logika kapitalisasi akan muncul berulang. Solusinya: buat **method helper** di dalam class.

```javascript
capitalizeFirst(value) {
  return value.charAt(0).toUpperCase() + value.slice(1);
}
```

Lalu getter dan setter bisa disederhanakan jadi:

```javascript
get firstName() {
  return this.capitalizeFirst(this._firstName);
}

set firstName(value) {
  this._firstName = this.capitalizeFirst(value);
}

get lastName() {
  return this.capitalizeFirst(this._lastName);
}

set lastName(value) {
  this._lastName = this.capitalizeFirst(value);
}
```

> ğŸ’¡ Gunakan `this.capitalizeFirst(...)` karena method itu ada di dalam class yang sama, jadi harus pakai `this`.

<a name="getter-untuk-full-name"></a>
## ğŸ·ï¸ Getter untuk Full Name

Ini contoh bagus lainnya untuk getter â€” menggabungkan dua properti jadi satu nilai baru.

```javascript
get fullName() {
  return `${this.firstName} ${this.lastName}`;
}
```

Cara pakainya:

```javascript
console.log(person1.fullName); // "Joe Smith"
```

`fullName` **bukan properti** yang tersimpan di object â€” melainkan dihitung on-the-fly setiap kali dipanggil. Kalau kamu lihat object di console, `fullName` tidak akan muncul sebagai properti, tapi ada di bagian **prototype** sebagai getter.
