<a name="masalah-sebelum-es2022"></a>
## 🤔 Masalah Sebelum ES2022

Dulu, JavaScript **tidak punya cara resmi** untuk membuat properti/method yang benar-benar privat di dalam sebuah class. Artinya, siapapun bisa mengakses dan mengubah data dari luar class — padahal kita nggak mau itu terjadi.

Solusi lama yang dipakai developer adalah **konvensi underscore** (`_balance`). Tapi ini cuma "perjanjian tidak tertulis" — tidak ada yang benar-benar mencegah orang mengaksesnya dari luar.

```javascript
// ❌ Cara lama: underscore hanya "konvensi", bukan benar-benar private
class Wallet {
  constructor() {
    this._balance = 0; // Siapapun masih bisa akses wallet._balance dari luar!
  }
}
```

Ada cara lain pakai `Symbol` atau `WeakMap`, tapi ribet. Nah, sekarang ada cara yang jauh lebih simpel dan resmi!

---

<a name="solusinya-private-fields"></a>
## ✨ Solusinya: Private Fields dengan `#`

Di **ES2022**, JavaScript akhirnya punya fitur bernama **Class Fields** (atau *Private Fields*). Caranya: ganti underscore dengan tanda **`#`** (number sign / pagar).

Bedanya dengan underscore? Ini **benar-benar private** — JavaScript akan langsung error kalau kamu coba akses dari luar class!

```javascript
// ✅ Cara baru: # membuat properti BENAR-BENAR tidak bisa diakses dari luar
class Wallet {
  #balance = 0; // Ini privat sungguhan!
}

const wallet = new Wallet();
console.log(wallet.#balance); // ❌ ERROR! "Private field '#balance' must be declared in an enclosing class"
```

Bahkan **VS Code langsung kasih warning** merah sebelum kamu jalankan kodenya — keren kan?

<a name="cara-menulis-private-fields"></a>
## 🏗️ Cara Menulis Private Fields

Ada hal unik dari private fields: **kamu deklarasikannya langsung di body class**, bukan di dalam `constructor` seperti biasanya.

```javascript
class Wallet {
  // 👇 Dideklarasikan langsung di sini, bukan di constructor!
  #balance = 0;
  #transactions = [];

  // ... methods di sini
}
```

Ini berbeda dari cara biasa yang pakai `this.balance = 0` di dalam `constructor`. Dengan `#`, kamu tulisnya langsung di level class.

<a name="private-methods"></a>
## 🔒 Private Methods Juga Bisa!

Bukan cuma properti, **method pun bisa dibuat private** dengan cara yang sama — cukup tambahkan `#` di depan nama method-nya.

```javascript
class Wallet {
  #balance = 0;
  #transactions = [];

  // Method ini PUBLIC — bisa dipanggil dari luar
  deposit(amount) {
    this.#processDeposit(amount); // Memanggil method private dari dalam class ✅
    this.#balance += amount;
  }

  // Method ini PRIVATE — hanya bisa dipanggil dari dalam class
  #processDeposit(amount) {
    console.log(`Depositing ${amount}`);
    this.#transactions.push({ type: 'deposit', amount });
  }
}

const wallet = new Wallet();
wallet.deposit(500);         // ✅ Boleh, ini public
wallet.#processDeposit(500); // ❌ ERROR! Ini private
```

> 💡 **Perhatikan:** `#processDeposit` dipanggil dari dalam method `deposit` yang ada di dalam class yang sama — itu **boleh**! Yang tidak boleh adalah memanggilnya dari luar class.

<a name="getter"></a>
## 🚪 Getter: Pintu Resmi untuk Akses Data

Karena `#balance` private, kamu nggak bisa akses langsung dari luar. Kalau coba akses `wallet.balance`, hasilnya `undefined` karena properti publik bernama `balance` memang tidak ada.

Solusinya? Buat **getter** — semacam "jendela resmi" yang membolehkan orang *melihat* nilainya, tapi tidak bisa mengubahnya sembarangan.

```javascript
class Wallet {
  #balance = 0;
  #transactions = [];

  // Getter untuk balance
  get balance() {
    return this.#balance; // Dari dalam class, #balance bisa diakses ✅
  }

  // Getter untuk transactions
  get transactions() {
    return this.#transactions;
  }
}

const wallet = new Wallet();
wallet.deposit(500);
console.log(wallet.balance); // ✅ Output: 500 (lewat getter)
```

<a name="kode-lengkap"></a>
## 🧪 Kode Lengkap dan Penjelasannya

Berikut kode lengkap class `Wallet` dengan semua konsep yang sudah dibahas:

```javascript
class Wallet {
  // 🔒 Private fields — hanya bisa diakses dari dalam class ini
  #balance = 0;
  #transactions = [];

  // ✅ PUBLIC: Bisa dipanggil dari luar
  deposit(amount) {
    this.#processDeposit(amount); // Panggil method private dari dalam
    this.#balance += amount;      // Update private field dari dalam
  }

  // ✅ PUBLIC: Bisa dipanggil dari luar
  withdraw(amount) {
    if (amount > this.#balance) {
      console.log('Not enough funds');
      return;
    }
    this.#processWithdraw(amount);
    this.#balance -= amount;
  }

  // 🔒 PRIVATE: Hanya bisa dipanggil dari dalam class
  #processDeposit(amount) {
    console.log(`Depositing ${amount}`);
    this.#transactions.push({ type: 'deposit', amount });
  }

  // 🔒 PRIVATE: Hanya bisa dipanggil dari dalam class
  #processWithdraw(amount) {
    console.log(`Withdrawing ${amount}`);
    this.#transactions.push({ type: 'withdraw', amount });
  }

  // 🚪 GETTER PUBLIC: Pintu resmi untuk melihat balance
  get balance() {
    return this.#balance;
  }

  // 🚪 GETTER PUBLIC: Pintu resmi untuk melihat daftar transaksi
  get transactions() {
    return this.#transactions;
  }
}

// === Penggunaan ===
const wallet = new Wallet();
wallet.deposit(500);
wallet.withdraw(100);
console.log(wallet.balance); // Output: 400
```

### 🎯 Apa yang Public dan apa yang Private?

| Yang **Public** (bisa diakses dari luar) | Yang **Private** (hanya di dalam class) |
|---|---|
| `deposit(amount)` | `#balance` |
| `withdraw(amount)` | `#transactions` |
| getter `balance` | `#processDeposit(amount)` |
| getter `transactions` | `#processWithdraw(amount)` |

Inilah yang disebut **API dari sebuah object** — antarmuka resmi yang boleh dipakai oleh kode di luar class.

<a name="ringkasan"></a>
## 📌 Ringkasan

- Sebelum ES2022, JavaScript tidak punya cara resmi untuk membuat properti benar-benar private
- Konvensi underscore (`_nama`) hanya perjanjian tidak tertulis, bukan perlindungan nyata
- Mulai ES2022, gunakan **`#`** di depan nama properti atau method untuk membuatnya **benar-benar private**
- Private fields dideklarasikan **langsung di body class**, bukan di dalam constructor
- Gunakan **getter** untuk memberi akses baca ke data private dari luar class
- VS Code bahkan langsung kasih error warning kalau kamu coba akses private field dari luar — super helpful! 🎉
