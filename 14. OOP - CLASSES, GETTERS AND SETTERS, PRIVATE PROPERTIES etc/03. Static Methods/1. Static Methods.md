## ğŸ¯ Pengenalan

**Static method** adalah method yang tersedia langsung di **class-nya**, bukan di *instance* (objek) yang dibuat dari class tersebut.

Kalau kamu pernah pakai bahasa lain yang punya konsep class (seperti Java, C#, atau Python), konsep ini pasti sudah familiar!

Bayangkan kita punya class `Rectangle` (persegi panjang).

### Method Biasa â€” butuh instance

Method `area()` menghitung luas dengan cara mengalikan `height` dan `width`. Karena dia butuh data dari objeknya, kita **wajib** bikin instance dulu sebelum bisa memanggilnya.

```javascript
class Rectangle {
  constructor(name, height, width) {
    this.name = name;
    this.height = height;
    this.width = width;
  }

  // Method biasa â€” butuh this.height dan this.width
  area() {
    return this.height * this.width;
  }
}

// Harus bikin instance dulu
const rect = new Rectangle('Rect', 10, 10);
console.log(rect.area()); // âœ… Output: 100
```

### Static Method â€” langsung pakai class-nya

Static method adalah method yang **tidak butuh instance**. Dia dipanggil langsung dari nama class-nya.

```javascript
class Rectangle {
  // ...

  // Static method â€” tidak butuh data dari instance
  static getClass() {
    return 'Rectangle';
  }
}

// Dipanggil langsung dari class, BUKAN dari instance
console.log(Rectangle.getClass()); // âœ… Output: Rectangle
```

<a name="kapan-pakai-static-method"></a>
## ğŸ¤” Kapan Pakai Static Method?

Gunakan static method ketika method tersebut **tidak bergantung pada data spesifik dari sebuah objek**.

Contoh kasus yang cocok:
- Method yang mengembalikan info tentang class itu sendiri (seperti nama server, nama class, dll.)
- **Class UI** yang bertugas menambah/menghapus elemen dari DOM â€” karena hanya ada satu UI, banyak method-nya bisa dibuat static.
