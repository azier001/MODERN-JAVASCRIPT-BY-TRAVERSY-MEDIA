## ğŸ¯ Pengenalan Classes

Class itu sebenarnya **bukan bagian inti dari JavaScript**. Fitur ini baru ditambahkan di update **ES6 (2015)**. Tapi walau begitu, class sangat berguna karena membuat kode kita jauh lebih rapi dan mudah dibaca.

> ğŸ’¡ **Yang penting diingat:** Di balik layar, class tetap bekerja dengan cara yang sama seperti sebelumnya â€” pakai **constructor function** dan **prototype**. Class hanya cara penulisan yang lebih enak aja!

Jadi kalau kamu bikin method di dalam class, method itu sebenarnya disimpan di **prototype**, bukan langsung di objek. Ini sama persis dengan cara lama.

<a name="membuat-class-pertama"></a>
## ğŸ—ï¸ Membuat Class Pertama

Untuk membuat class, kita pakai keyword `class` diikuti nama classnya, lalu buka kurung kurawal `{}`.

```javascript
class Rectangle {
  // isi class di sini
}
```

Setelah class dibuat, kita bisa **instantiate** (membuat objek baru dari class itu) dengan keyword `new`:

```javascript
const square = new Rectangle('Square', 20, 20);
```

<a name="constructor-method"></a>
## ğŸ”§ Constructor Method

Di dalam class, hal pertama yang kita tulis adalah **constructor**. Constructor ini mirip banget dengan constructor function biasa â€” bedanya cuma dia ada *di dalam* class.

```javascript
class Rectangle {
  constructor(name, width, height) {
    this.name = name;
    this.width = width;
    this.height = height;
  }
}
```

### Cara kerja constructor:
- Constructor **otomatis dijalankan** setiap kali kamu membuat objek baru pakai `new`
- Kalau kamu buat 2 objek baru, constructor akan jalan **2 kali**
- Di dalam constructor, `this` merujuk ke **objek yang lagi dibuat itu**

```javascript
const square = new Rectangle('Square', 20, 20);   // constructor jalan 1x
const square2 = new Rectangle('Square2', 10, 15); // constructor jalan lagi
```

## âš™ï¸ Menambahkan Methods

Untuk menambah method ke dalam class, cukup tulis nama methodnya langsung â€” **tanpa keyword `function`**. Kalau pakai `function`, akan error!

```javascript
class Rectangle {
  constructor(name, width, height) {
    this.name = name;
    this.width = width;
    this.height = height;
  }

  // âœ… Benar â€” tanpa keyword function
  area() {
    return this.height * this.width;
  }

  // âŒ Salah â€” akan error!
  // function area() { ... }
}
```

Di dalam method, kita tetap bisa pakai `this` untuk mengakses properti objek.

### Contoh 3 method pada class Rectangle:

```javascript
class Rectangle {
  constructor(name, width, height) {
    this.name = name;
    this.width = width;
    this.height = height;
  }

  // Menghitung luas
  area() {
    return this.height * this.width;
  }

  // Menghitung keliling
  perimeter() {
    return 2 * (this.width + this.height);
  }

  // Cek apakah persegi (bukan persegi panjang)
  isSquare() {
    return this.width === this.height; // returns true atau false
  }
}
```

Menggunakannya:

```javascript
const square = new Rectangle('Square', 20, 20);

console.log(square.area());       // 400
console.log(square.perimeter());  // 80
console.log(square.isSquare());   // true
```

> ğŸ” **Ingat:** Semua method ini tersimpan di **prototype** objek, bukan langsung di objek itu sendiri. Kamu bisa cek dengan: `Object.getPrototypeOf(square)`

<a name="mengakses-method-dari-method-lain"></a>
## ğŸ”— Mengakses Method dari Method Lain

Selain mengakses **properti** dengan `this`, kita juga bisa mengakses **method lain** di dalam class yang sama menggunakan `this`!

```javascript
class Rectangle {
  constructor(name, width, height) {
    this.name = name;
    this.width = width;
    this.height = height;
  }

  area() {
    return this.height * this.width;
  }

  // Method ini memanggil method area() dari dalam class
  logArea() {
    console.log('Rectangle Area: ' + this.area()); // pakai this.area()
  }
}

const square = new Rectangle('Square', 20, 20);
square.logArea(); // Output: Rectangle Area: 400
```

### Kenapa ini berguna? ğŸ¤”

Ini berkaitan dengan konsep **abstraction** â€” pengguna class tidak perlu tahu cara kerja di dalamnya. Bayangkan seperti TV:
- Kamu cuma perlu tahu tombol power dan channel
- Kamu **nggak perlu tahu** komponen elektronik di dalamnya

Begitu juga class â€” bisa punya banyak method "internal" yang hanya dipakai di dalam class itu sendiri, dan pengguna luar tidak perlu tahu soal itu.
