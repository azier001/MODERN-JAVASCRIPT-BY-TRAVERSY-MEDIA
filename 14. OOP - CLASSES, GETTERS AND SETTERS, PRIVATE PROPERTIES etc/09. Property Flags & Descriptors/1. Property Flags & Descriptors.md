<a name="apa-itu-property-flags"></a>
## 🔍 Apa itu Property Flags?

Di JavaScript, setiap properti dalam sebuah objek punya semacam **"pengaturan izin"** yang disebut **property flags**. Bayangkan seperti pengaturan privasi — kamu bisa mengontrol apakah sebuah properti boleh diubah, dihapus, atau bahkan dilihat saat di-loop.

> 💡 **Penting:** Saat kamu membuat objek baru, semua flag ini secara default diset ke `true`.

---

<a name="tiga-flag-yang-tersedia"></a>
## 🚩 Tiga Flag yang Tersedia

| Flag | Nilai Default | Artinya |
|------|:---:|---------|
| `writable` | `true` | Kalau `true`, nilai properti bisa diubah |
| `configurable` | `true` | Kalau `true`, properti bisa dihapus dan flag-nya bisa dimodifikasi |
| `enumerable` | `true` | Kalau `true`, properti akan muncul saat di-loop (misal: `for...in`) |

> ⚠️ **Hati-hati dengan `configurable: false`!** Kalau kamu set ini ke `false`, kamu **tidak bisa** mengubah flag-flag tersebut kembali. Permanen!

### Contoh nyata: `Math.PI`

Salah satu contoh yang sudah ada di JavaScript adalah `Math.PI`. Properti ini punya semua flag yang di-set ke `false`, jadi nilainya tidak bisa diubah sama sekali:

```javascript
// Coba ubah Math.PI... tidak akan berhasil!
Math.PI = 4;
console.log(Math.PI); // tetap 3.141592653589793
```

<a name="melihat-flags-dengan-descriptor"></a>
## 🔬 Melihat Flags dengan Descriptor

Untuk melihat flag dari sebuah properti, gunakan method `Object.getOwnPropertyDescriptor()`. Method ini menerima dua argumen: **objek** dan **nama propertinya**.

### Contoh: Cek descriptor `Math.PI`

```javascript
let descriptor = Object.getOwnPropertyDescriptor(Math, 'PI');
console.log(descriptor);
```

Output yang akan kamu lihat:
```
{
  value: 3.141592653589793,
  writable: false,       // tidak bisa diubah
  enumerable: false,     // tidak muncul saat di-loop
  configurable: false    // tidak bisa dihapus atau dimodifikasi
}
```

### Contoh: Cek descriptor objek buatan sendiri

Sekarang bandingkan dengan objek yang kita buat sendiri:

```javascript
const rectObj = {
  name: 'Rectangle 1',
  width: 10,
  height: 10,
};

let descriptor = Object.getOwnPropertyDescriptor(rectObj, 'name');
console.log(descriptor);
```

Output:
```
{
  value: 'Rectangle 1',
  writable: true,       // bisa diubah ✅
  enumerable: true,     // muncul saat di-loop ✅
  configurable: true    // bisa dihapus/dimodifikasi ✅
}
```

Semua `true` karena itulah **nilai default** saat kita membuat objek.

<a name="mengubah-flags-dengan-defineproperty"></a>
## 🛠️ Mengubah Flags dengan `defineProperty`

Untuk mengubah flags, gunakan `Object.defineProperty()`. Method ini menerima **tiga argumen**:
1. Objeknya
2. Nama properti yang ingin diubah flagnya
3. Objek berisi flag-flag yang ingin diubah

```javascript
const rectObj = {
  name: 'Rectangle 1',
  width: 10,
  height: 10,
};

// Kunci properti 'name' agar tidak bisa diubah, dihapus, atau di-loop
Object.defineProperty(rectObj, 'name', {
  writable: false,
  configurable: false,
  enumerable: false,
});
```

### Efeknya: Tidak bisa diubah

```javascript
rectObj.name = 'New Name';  // mencoba mengubah...
console.log(rectObj);
// { name: 'Rectangle 1', width: 10, height: 10 }
// nama tetap 'Rectangle 1', tidak berubah!
```

### Efeknya: Tidak bisa dihapus

```javascript
delete rectObj.name;        // mencoba menghapus...
console.log(rectObj);
// { name: 'Rectangle 1', width: 10, height: 10 }
// properti 'name' masih ada!

// Tapi properti lain yang tidak dikunci bisa dihapus:
delete rectObj.width;
console.log(rectObj);
// { name: 'Rectangle 1', height: 10 }  ← width berhasil dihapus
```

---

<a name="efek-enumerable-false-pada-looping"></a>
## 🔄 Efek `enumerable: false` pada Looping

Saat `enumerable` di-set ke `false`, properti tersebut akan **disembunyikan** dari proses looping seperti `for...of` dengan `Object.entries()`.

```javascript
// Loop semua properti rectObj
for (let [key, value] of Object.entries(rectObj)) {
  console.log(`${key}: ${value}`);
}
```

Output (saat `enumerable: false` pada `name`):
```
width: 10
height: 10
// 'name' tidak muncul! 👻
```

Jika `enumerable` diubah kembali ke `true`:
```
name: Rectangle 1
width: 10
height: 10
// sekarang 'name' muncul lagi ✅
```

<a name="melihat-semua-descriptor-sekaligus"></a>
## 📦 Melihat Semua Descriptor Sekaligus

Kalau mau melihat descriptor untuk **semua properti** sekaligus (bukan cuma satu), gunakan versi plural-nya: `Object.getOwnPropertyDescriptors()`. Cukup pass objeknya saja, tanpa perlu menyebutkan nama properti tertentu.

```javascript
console.log(Object.getOwnPropertyDescriptors(rectObj));
```

Output yang dihasilkan:
```
{
  name: {
    value: 'Rectangle 1',
    writable: false,        // ← karena kita ubah tadi
    enumerable: false,      // ← karena kita ubah tadi
    configurable: false     // ← karena kita ubah tadi
  },
  width: {
    value: 10,
    writable: true,
    enumerable: true,
    configurable: true
  },
  height: {
    value: 10,
    writable: true,
    enumerable: true,
    configurable: true
  }
}
```

Terlihat jelas perbedaannya — properti `name` punya semua flag `false`, sementara `width` dan `height` tetap `true` karena kita tidak mengubahnya.

---

> 📌 **Ringkasan:** Dengan property flags, kita bisa mengontrol akses ke properti objek secara granular — mau read-only, tidak bisa dihapus, atau bahkan disembunyikan dari loop. Method kuncinya adalah `defineProperty` untuk mengubah, dan `getOwnPropertyDescriptor(s)` untuk mengecek.
