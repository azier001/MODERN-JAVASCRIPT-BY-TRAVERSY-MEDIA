## ğŸ”€ Langkah 3: Membuat Switch Statement untuk Router

### ğŸ¯ Tujuan
Menggunakan `switch` untuk memeriksa halaman mana yang sedang aktif.

### ğŸ’» Kode

Tambahkan kode ini **di dalam fungsi `init()`**:

```javascript

      console.log('Home');
      break;
  }

```

### ğŸ“– Penjelasan

- `switch (global.currentPage)` â†’ Memeriksa nilai `currentPage`
- `case '/'` â†’ Jika halaman adalah root (`/`)
- `console.log('Home')` â†’ Tampilkan "Home" di console
- `break` â†’ Hentikan pengecekan setelah case ini

### ğŸ¤” Mengapa Pakai Switch?

Switch lebih bersih dan mudah dibaca dibanding banyak `if-else` statement:

```javascript

  console.log('Home');
} else if (global.currentPage === '/shows.html') {
  console.log('Shows');
} else if (global.currentPage === '/movie-details.html') {
  console.log('Movie Details');
}

// âœ… Cara dengan switch (lebih bersih)

  case '/':
    console.log('Home');
    break;
  case '/shows.html':
    console.log('Shows');
    break;
  case '/movie-details.html':
    console.log('Movie Details');
    break;
```

<a name="langkah-9-menangani-multiple-conditions"></a>
## ğŸ”„ Langkah 9: Menangani Multiple Conditions

### ğŸ¯ Tujuan
Membuat satu case menangani dua kondisi berbeda (untuk homepage).

### â“ Masalah

Homepage bisa diakses dengan dua cara:
- `/` (root)
- `/index.html` (halaman index)

Saat ini, jika kita akses `/index.html`, tidak ada yang terjadi!

### ğŸ’» Solusi

Gunakan **multiple case** tanpa `break` di antaranya:

```javascript
function init() {
  switch (global.currentPage) {
    case '/':
    case '/index.html':
      console.log('Home');
      break;
    
    case '/shows.html':
      console.log('Shows');
      break;
    
    case '/movie-details.html':
      console.log('Movie Details');
      break;
    
    case '/tv-details.html':
      console.log('TV Details');
      break;
    
    case '/search.html':
      console.log('Search');
      break;
  }
}
```

### ğŸ“– Penjelasan

Ketika tidak ada `break` setelah case pertama, JavaScript akan "jatuh" ke case berikutnya. Ini disebut **fall-through behavior**.

```javascript
case '/':           // Jika ini cocok...
case '/index.html': // ...atau ini cocok...
  console.log('Home'); // ...jalankan kode ini
  break;            // ...lalu keluar
```

### ğŸ§ª Testing

Coba akses:
- `http://localhost/` â†’ Output: `Home` âœ…
- `http://localhost/index.html` â†’ Output: `Home` âœ…

Kedua URL sekarang menghasilkan output yang sama!
