<a name="operator-perbandingan"></a>
## ğŸ“Š Operator Perbandingan

Berikut adalah operator yang bisa digunakan dalam IF statement:

| **Operator** | **Definisi** | **Contoh** | **Hasil** |
|:---:|:---:|:---:|:---:|
| `<` | Less than (Kurang dari) | `10 < 5` | False |
| `>` | Greater than (Lebih besar dari) | `10 > 5` | True |
| `<=` | Less than or equal to | `10 <= 10` | True |
| `>=` | Greater than or equal to | `10 >= 5` | True |
| `==` | Equal to (Sama dengan) | `10 == '10'` | True |
| `!=` | Not equal to (Tidak sama dengan) | `10 != 5` | True |
| `===` | Equal to (including type) | `10 === '10'` | False |
| `!==` | Not equal to (including type) | `10 !== '10'` | True |

### ğŸ’¡ Perbedaan `==` vs `===`

- **`==` (Double Equal)**: Hanya membandingkan nilai, **tidak peduli tipe data**
  ```javascript
  10 == '10'  // true (nilai sama, tipe diabaikan)
  ```

- **`===` (Triple Equal)**: Membandingkan nilai **DAN tipe data**
  ```javascript
  10 === '10'  // false (nilai sama, tapi tipe berbeda: number vs string)
  ```

> âš ï¸ **Best Practice**: Gunakan `===` untuk menghindari bug yang tidak terduga!

## ğŸ”’ Block Scope dalam IF Statement

IF Statement memiliki **scope sendiri**. Variable yang dideklarasikan di dalam IF tidak bisa diakses dari luar.

### âœ… Contoh dengan `let` dan `const`:

```javascript

```

**Output:**


**Penjelasan:**
- Variable `z` dideklarasikan dengan `const` di dalam IF block
- `z` **HANYA** bisa diakses di dalam block tersebut
- Mencoba akses `z` di luar block = **ERROR** âŒ

### âš ï¸ Contoh dengan `var` (Tidak Direkomendasikan):


**Output:**

**Penjelasan:**
- `var` **TIDAK** memiliki block scope
- `var` hanya punya function scope
- Variable `z` bisa diakses di luar IF block
- **Ini bukan best practice!** âš ï¸

> ğŸ’¡ **Best Practice**: Selalu gunakan `let` atau `const`, JANGAN gunakan `var`!
