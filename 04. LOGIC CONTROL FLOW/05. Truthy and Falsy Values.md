ğŸ’¡ **Penjelasan `!isNaN()`:**

<a name="loose-vs-strict-equality"></a>
### 4ï¸âƒ£ Loose vs Strict Equality

#### ğŸ”„ **Loose Equality (`==`):**
Membandingkan nilai TANPA mengecek tipe data (melakukan type coercion).

#### ğŸ”’ **Strict Equality (`===`):**
Membandingkan nilai DAN tipe data (tidak melakukan type coercion).

### ğŸ“ **Contoh Perbandingan:**

```javascript

console.log('' == 0);              // true (keduanya falsy)
console.log(null == undefined);    // true (keduanya falsy)

// âœ… Strict Equality (===) - Lebih aman!
console.log(false === 0);          // false (beda tipe)
console.log('' === 0);             // false (beda tipe)
console.log(null === undefined);   // false (beda tipe)
```

### ğŸ¯ **Contoh Kasus Nyata:**

```javascript

const targetValue = 0;  // Nilai yang dicari (number)

// âŒ Menggunakan == (loose)
if (userInput == targetValue) {
  console.log("Match!"); // âœ… Akan match (tapi mungkin tidak sesuai harapan)
}

// âœ… Menggunakan === (strict)
if (userInput === targetValue) {
  console.log("Match!");
} else {
  console.log("Not match!"); // âœ… Lebih jelas, tidak match karena beda tipe
}

// ğŸ”§ Solusi yang benar: Convert dulu

  console.log("Match!"); // âœ… Benar dan jelas!
}
```

### ğŸ’¡ **Rekomendasi:**
Selalu gunakan `===` dan `!==` kecuali ada alasan khusus untuk menggunakan `==` dan `!=`.
