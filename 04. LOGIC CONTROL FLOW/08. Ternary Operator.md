## ğŸ“– Apa itu Ternary Operator?

**Ternary Operator** adalah operator kondisional yang terdiri dari **3 bagian** (makanya disebut "ternary" yang artinya "tiga"). Operator ini menggunakan **tanda tanya `?`** dan **titik dua `:`**.

### ğŸ”‘ Fungsi Utama:
Menggantikan struktur **if-else** yang sederhana menjadi **satu baris kode**.

#### **Cara 1: Ternary dengan `null`**

- Jika `auth` true â†’ jalankan console.log

- Jika `auth` true â†’ jalankan console.log
- Jika `auth` false â†’ tidak melakukan apa-apa
- **SAMA PERSIS** dengan cara 1, tapi **lebih singkat**!

### ğŸ’¡ Kapan Menggunakan?
- âœ… Gunakan `&&` jika **hanya butuh aksi saat true**
- âœ… Sangat berguna di **React JSX**
- âœ… Membuat kode lebih **clean** dan **compact**

<a name="kesimpulan"></a>
## ğŸ“ Kesimpulan

### ğŸ“Œ Ringkasan Ternary Operator:

| Tipe | Syntax | Kapan Digunakan |
|------|--------|-----------------|
| **Basic Ternary** | `kondisi ? true : false` | Kondisi sederhana dengan 2 hasil |
| **Variable Assignment** | `const x = kondisi ? A : B` | Menyimpan nilai kondisional |
| **Multiple Statements** | `kondisi ? (a, b) : (c, d)` | Butuh beberapa aksi sekaligus |
| **Shorthand (&&)** | `kondisi && aksi` | Hanya butuh aksi saat true |

### âœ¨ Keuntungan Ternary Operator:

- âœ… **Lebih Singkat**: 1 baris vs 5-6 baris
- âœ… **Lebih Clean**: Kode lebih rapi dan mudah dibaca
- âœ… **Lebih Modern**: Digunakan di framework modern (React, Vue)
- âœ… **Lebih Efisien**: Mengurangi jumlah kode yang ditulis

### ğŸ¯ Tips Penggunaan:

1. **Gunakan untuk kondisi SEDERHANA** - Jangan terlalu kompleks
2. **Prioritaskan Readability** - Jika terlalu panjang, pakai if-else saja
3. **Latihan Konsisten** - Makin sering pakai, makin terbiasa
4. **Kombinasikan dengan &&** - Untuk kondisi tanpa else
