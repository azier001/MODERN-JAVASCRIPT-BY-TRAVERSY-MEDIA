<a name="apa-itu-refactoring"></a>
## ğŸ“– Apa itu Refactoring?

**Refactoring** adalah proses memperbaiki struktur kode **tanpa mengubah fungsinya**.

### Analogi Sederhana ğŸ 

Bayangkan kamu punya kamar yang berantakan:
- Baju berserakan
- Buku tidak tertata
- Barang-barang tidak punya tempat tetap

**Refactoring** = Merapikan kamar:
- Baju dilipat di lemari
- Buku disusun di rak
- Setiap barang punya tempat

Kamar tetap **kamar yang sama**, tapi sekarang **lebih rapi dan mudah dicari barangnya**.

## ğŸ“¦ Perubahan 2: `getItemsFromStorage` â†’ `loadItemsFromStorage`


**Masalah**:
- Nama kurang deskriptif
- Kode terlalu panjang (9 baris)
- If-else bertele-tele

---

### âœ… Kode Baru

```javascript

  const storedItems = localStorage.getItem(STORAGE_KEY); // ğŸ‘ˆ Pakai konstanta
  
  // âœ¨ DIUBAH: If-else â†’ Ternary operator
  return storedItems ? JSON.parse(storedItems) : [];
  // ğŸ‘† Kalau ada data, parse JSON. Kalau tidak, return array kosong

```

**Hasil**: Dari 9 baris jadi **3 baris**! ğŸ‰

---

### ğŸ“ Penjelasan

#### 1. Kenapa Nama Diubah?

| Nama Lama | Nama Baru | Alasan |
|-----------|-----------|--------|
| `getItemsFromStorage` | `loadItemsFromStorage` | `load` lebih umum untuk "memuat data" |

**Analogi**: 
- `get` = ambil sesuatu
- `load` = muat/ambil data dari suatu tempat (storage, database, dll)

Dalam dunia programming, `load` lebih sering dipakai untuk operasi mengambil data.

---

#### 2. Apa itu Ternary Operator?

**Ternary operator** adalah cara singkat menulis if-else:

```javascript
// Format:
kondisi ? nilai_jika_benar : nilai_jika_salah

// Contoh:
const umur = 17;
const status = umur >= 18 ? 'Dewasa' : 'Anak-anak';
// Jika umur >= 18, status = 'Dewasa'
// Jika tidak, status = 'Anak-anak'
```

**Dalam kode kita:**

```javascript
// âŒ Kode lama (5 baris)
if (localStorage.getItem('items') === null) {
  itemsFromStorage = [];
} else {
  itemsFromStorage = JSON.parse(localStorage.getItem('items'));
}

// âœ… Kode baru (1 baris)
return storedItems ? JSON.parse(storedItems) : [];
```

**Penjelasan**:
- `storedItems` â†’ kondisi yang dicek
- `?` â†’ kalau kondisi benar (storedItems ada)
- `JSON.parse(storedItems)` â†’ lakukan ini
- `:` â†’ kalau kondisi salah (storedItems null)
- `[]` â†’ return array kosong

---

#### 3. Kenapa Lebih Baik?

âœ… **Lebih Ringkas**: 9 baris â†’ 3 baris
âœ… **Lebih Modern**: Ternary operator adalah standar JavaScript modern
âœ… **Lebih Mudah Dibaca**: Setelah terbiasa, lebih cepat dipahami
âœ… **Menghindari Typo**: Pakai konstanta `STORAGE_KEY`

**Lebih masuk akal**: Tunggu DOM ready dulu, baru jalankan inisialisasi.
