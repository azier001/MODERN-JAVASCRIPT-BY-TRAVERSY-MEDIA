<a name="tips-best-practices"></a>
## ğŸ’¡ Tips & Best Practices

### âœ… DO (Lakukan):

1. **Commit sering** - Jangan tunggu sampai banyak perubahan
2. **Pesan commit yang jelas** - "added login feature" bukan "update"
3. **Push setiap hari** - Backup code kamu secara rutin
4. **Buat .gitignore** - Jangan push file yang tidak perlu (node_modules, .env)
5. **Baca pesan error** - Git error message sangat informatif

### âŒ DON'T (Hindari):

1. **Jangan commit file sensitive** - Password, API keys, .env
2. **Jangan push code yang error** - Test dulu sebelum push
3. **Jangan commit terlalu besar** - Commit setiap fitur, bukan sekaligus
4. **Jangan abaikan .gitignore** - File besar/tidak perlu jangan di-push

### ğŸ“… Kapan Harus Commit?

**Good commit points:**
- âœ… Selesai membuat fitur baru
- âœ… Memperbaiki bug
- âœ… Refactoring code
- âœ… Update documentation
- âœ… Sebelum coba hal yang risky (bisa rollback jika gagal)

**Contoh workflow:**
```bash
# Selesai buat login page
git add .
git commit -m "create login page with validation"
git push

# Selesai fix bug
git add .
git commit -m "fix navbar overlap on mobile"
git push

# Update styling
git add .
git commit -m "improve button hover effects"
git push
```
