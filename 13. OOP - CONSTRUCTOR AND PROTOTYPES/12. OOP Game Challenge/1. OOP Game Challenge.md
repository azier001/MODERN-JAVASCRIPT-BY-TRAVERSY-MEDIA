<a name="constructor-function"></a>
## ğŸ—ï¸ Membuat Constructor Function

Constructor function itu kayak "cetakan" buat bikin object baru. Setiap kali kita mau bikin player baru, kita tinggal panggil cetakan ini.

```js
function Player(name) {
  this.name = name;   // nama player dari luar
  this.lvl = 1;       // level awal selalu 1
  this.points = 0;    // poin awal selalu 0
}
```

> ğŸ’¡ `this` di sini merujuk ke object yang sedang dibuat. Jadi setiap player punya `name`, `lvl`, dan `points` masing-masing.

Untuk membuat player baru, gunakan keyword `new`:

```js
const player1 = new Player('Bob');
const player2 = new Player('Alice');
```

Kalau kamu `console.log(player1)`, hasilnya bakal ada `name`, `lvl`, dan `points` â€” plus di prototype-nya ada method `gainXp` dan `describe`.

<a name="prototype-method"></a>
## âš™ï¸ Menambahkan Method ke Prototype

Method yang kita taruh di **prototype** bisa dipakai oleh semua object yang dibuat dari constructor yang sama. Hemat memori karena tidak diduplikasi tiap object.

### Method `gainXp` â€” Nambah Poin

```js
Player.prototype.gainXp = function (xp) {
  this.points += xp;  // tambahkan xp ke poin yang sudah ada (bukan ganti!)

  if (this.points >= 10) {
    this.lvl++;         // naik level
    this.points -= 10;  // kurangi poin 10
  }
};
```

> âš ï¸ Perhatikan `+=` bukan `=`. Kalau pakai `=`, poin lama akan terganti. Kita mau **menambahkan**, bukan **mengganti**.

**Logika level up:**
- Kalau total poin sudah â‰¥ 10 â†’ level naik 1
- Poin dikurangi 10 (seperti "reset ke sisa")

### Method `describe` â€” Tampilkan Statistik Player

```js
Player.prototype.describe = function () {
  return `${this.name} is level ${this.lvl} with ${this.points} experience points`;
};
```

> ğŸ’¡ Pakai **template literal** (backtick `` ` ``) supaya bisa sisipkan variabel langsung dengan `${}`.

<a name="abstraksi"></a>
## ğŸ­ Konsep Abstraksi

Ini bonus insight menarik dari video! **Abstraksi** artinya menyembunyikan hal-hal yang tidak perlu diketahui pengguna.

Contohnya, method `describe` sebenarnya bisa dijadikan **internal** â€” dipanggil otomatis di dalam `gainXp`, bukan oleh pengguna secara langsung:

```js
Player.prototype.gainXp = function (xp) {
  this.points += xp;

  if (this.points >= 10) {
    this.lvl++;
    this.points -= 10;
  }

  console.log(this.describe()); // otomatis tampil setiap kali gainXp dipanggil
};
```

Dengan cara ini:
- `gainXp` â†’ dipakai pengguna âœ…
- `describe` â†’ dipakai *internal* saja, tidak perlu diekspos ke pengguna âœ…

> ğŸ’¡ Kalau kamu bikin dokumentasi untuk object ini, kamu cukup dokumentasikan `gainXp` saja. `describe` cukup jadi "urusan dalam".

Ini juga berkaitan dengan konsep **encapsulation** â€” mengelompokkan data dan logika ke dalam satu unit yang terorganisir.
