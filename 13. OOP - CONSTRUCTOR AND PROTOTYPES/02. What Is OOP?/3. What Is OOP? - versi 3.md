## ğŸ“¦ Apa itu Object?

Object adalah **potongan kode yang mandiri** yang berisi **kumpulan properti**.

Biar nggak bingung, ini breakdown-nya:

- **Properti** = pasangan antara *nama (key)* dan *nilai (value)*
- Kalau nilai dari sebuah properti adalah fungsi, maka itu disebut **method**

Jadi intinya:

```
Object = kumpulan properti + method
```

### Contoh dari kehidupan nyata ğŸ 

Kalau kamu bikin **aplikasi blog**, entity-entity ini bisa jadi object:
- `Post` â€” data artikel beserta fungsi untuk membuat/edit post
- `Comment` â€” komentar dengan fungsi untuk menghapus/reply
- `User` â€” user dengan fungsi login/logout

Kalau **e-commerce**:
- `Product`, `Order`, `Cart`, `Customer`

Kalau **HR app**:
- `Employee`, `Department`, `PaySlip`

> ğŸ“Œ Intinya: **hampir semua "entitas" di aplikasimu bisa dijadikan object!**

### Object yang sudah kita kenal ğŸ”

Sebenarnya kita sudah sering pakai object bawaan JavaScript, contohnya:
- `Math` â†’ `Math.random()`, `Math.floor()`
- `Date` â†’ `new Date()`

Bahkan string dan number, walau sebenarnya bukan object (*primitives*), bisa diperlakukan seperti object. Misalnya saat kamu pakai `.toUpperCase()` pada sebuah string, JavaScript diam-diam membuat **wrapper object** sementara di balik layar.

## â° Kapan Pakai OOP?

OOP **bukan** solusi untuk semua masalah. Ini panduannya:

### âœ… Pakai OOP kalau:
- Aplikasimu **kompleks** dan punya banyak bagian
- Kamu perlu **banyak instance** dari object yang sama (misalnya banyak user, banyak produk)
- Kamu **kerja dalam tim** â€” OOP membantu kode lebih terstruktur dan mudah dibagi

### âŒ Jangan dipaksakan OOP kalau:
- Aplikasinya **sederhana**
- Satu script pendek sudah cukup

### ğŸŒŸ Kenapa OOP populer?
1. Kode lebih **mudah dipahami** dan **mudah di-maintain**
2. Mencegah **spaghetti code** (kode yang amburadul dan susah diikuti)
3. Sangat **fleksibel** untuk aplikasi yang kompleks
4. Kode bisa **reuse** dengan mudah
