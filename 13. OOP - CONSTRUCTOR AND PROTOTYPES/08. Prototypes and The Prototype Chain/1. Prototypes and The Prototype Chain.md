<a name="apa-itu-prototype"></a>
## üéØ Apa Itu Prototype?

Banyak tutorial bikin topik ini keliatan rumit padahal sebenernya simpel. Kalau udah ngerti object, property, dan method ‚Äî prototype itu cuma satu langkah lagi.

> **Prototype** adalah object spesial tempat kita bisa naruh method dan property tambahan yang bisa dipakai (di-*share*) oleh semua instance dari sebuah constructor function.

Jadi kalau kita bikin banyak object dari satu constructor yang sama, mereka semua bisa akses method yang ada di prototype ‚Äî **tanpa harus copy-paste method itu ke tiap object**.

---

<a name="melihat-prototype-di-console"></a>
## üîç Melihat Prototype di Console

Kita mulai dari constructor function `Rectangle` dan buat satu instance-nya:

```javascript
function Rectangle(name, width, height) {
  this.name = name;
  this.width = width;
  this.height = height;
  this.area = function () {
    return this.width * this.height;
  };
}

const rect = new Rectangle('Rect', 10, 10);
console.log(rect);
```

Kalau kamu buka hasilnya di browser console, kamu bakal lihat dua hal:

1. **Properties langsung** ‚Üí `name`, `width`, `height`, `area` (yang kita definisikan di dalam constructor)
2. **`[[Prototype]]`** ‚Üí object tersembunyi yang otomatis ada

Di dalam `[[Prototype]]` itu, by default udah ada **constructor** ‚Äî yaitu fungsi `Rectangle` itu sendiri (yang menerima `name`, `width`, `height`).

<a name="prototype-chain"></a>
## ‚õìÔ∏è Prototype Chain

Karena prototype itu sendiri adalah sebuah **object**, dia juga punya prototype-nya sendiri. Inilah yang disebut **prototype chain** (rantai prototype).

Visualisasinya kira-kira begini:

```
rect
  ‚îî‚îÄ [[Prototype]] ‚Üí Rectangle.prototype
        ‚îî‚îÄ [[Prototype]] ‚Üí Object.prototype
              ‚îî‚îÄ [[Prototype]] ‚Üí null
```

Nah, di `Object.prototype` inilah ada method-method bawaan seperti:
- `toString()`
- `valueOf()`
- `isPrototypeOf()`

Karena `rect` ada di rantai ini, kita bisa pakai method tersebut langsung:

```javascript
// toString() ada di Object.prototype, bukan di rect langsung
// tapi tetap bisa dipanggil karena prototype chain
console.log(rect.toString()); // "[object Object]"
```

**Cara JavaScript nyari method:** Ketika kamu panggil `rect.toString()`, JS bakal:
1. Cek langsung di object `rect` ‚Üí tidak ada
2. Cek di `Rectangle.prototype` ‚Üí tidak ada
3. Cek di `Object.prototype` ‚Üí **ketemu!** ‚úÖ

<a name="prototype-pada-array--string"></a>
## üì¶ Prototype pada Array & String

Konsep yang sama berlaku untuk tipe data bawaan JavaScript.

### Array

```javascript
const arr = [];
console.log(arr);
```

Array punya **Array.prototype** yang isinya semua method yang sering kita pakai:

| Method | Lokasi |
|--------|--------|
| `push` | Array.prototype |
| `pop` | Array.prototype |
| `filter` | Array.prototype |
| `map` | Array.prototype |

Makanya di MDN docs, method-method itu selalu ditulis sebagai `Array.prototype.filter`, `Array.prototype.map`, dst. Karena memang lokasinya di sana!

Dan sama seperti sebelumnya, `Array.prototype` juga **mewarisi** `Object.prototype`, jadi kita bisa pakai `toString()` di array juga.

### String

```javascript
const str = new String('hello');
console.log(str);
```

String punya **String.prototype** dengan method seperti:
- `charAt()`
- `endsWith()`
- `toUpperCase()`
- `toLowerCase()`

Dan String.prototype juga mewarisi Object.prototype ‚Äî polanya selalu sama! üîÑ

<a name="objectgetprototypeof"></a>
## üõ†Ô∏è Object.getPrototypeOf()

Ada cara buat langsung lihat prototype dari sebuah object pakai method bawaan:

```javascript
// Object.getPrototypeOf() ‚Üí untuk melihat prototype dari sebuah object
console.log(Object.getPrototypeOf(rect));
```

Hasilnya akan menampilkan prototype dari `rect`, yang sekarang isinya cuma `constructor` karena kita belum tambahkan method apapun ke `Rectangle.prototype`.

> üí° **Catatan:** Method-method seperti `Object.getPrototypeOf()`, `Object.assign()`, `Object.entries()`, `Object.keys()`, `Object.values()` semuanya ada di **object constructor** (`Object` itu sendiri), bukan di `Object.prototype`.

<a name="kenapa-ini-penting"></a>
## üöÄ Kenapa Ini Penting?

Lihat lagi kode `Rectangle` kita ‚Äî method `area` sekarang ada **di dalam constructor**:

```javascript
this.area = function () {
  return this.width * this.height;
};
```

Artinya setiap kali kita buat instance baru, JavaScript akan **membuat ulang fungsi `area`** untuk setiap object. Padahal logikanya selalu sama ‚Äî width √ó height.

**Solusinya?** Taruh `area` di **prototype** ‚Äî cukup satu definisi, bisa dipakai semua instance. Ini yang akan dibahas di video selanjutnya.

```
// Konsepnya nanti akan jadi seperti ini:
Rectangle.prototype.area = function() {
  return this.width * this.height;
}
// ‚Üí Satu fungsi, dipakai semua instance Rectangle ‚úÖ
```

---

> üìù **Ringkasan:** Prototype = object tersembunyi tempat method bersama. Prototype chain = jalur pencarian method dari object ‚Üí prototype ‚Üí Object.prototype ‚Üí null.
