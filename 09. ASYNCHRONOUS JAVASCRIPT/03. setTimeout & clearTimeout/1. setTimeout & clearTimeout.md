> ğŸ’¡ **Penjelasan:** Variable `timerId` menyimpan ID dari timer yang sedang berjalan

<a name="cara-membatalkan-timeout"></a>
### âŒ Cara Membatalkan Timeout

Untuk membatalkan sebuah timeout yang sedang berjalan, kita menggunakan `clearTimeout()` dengan memasukkan Timer ID.

**Contoh:**

> ğŸ’¡ **Penjelasan:** 
> - Ketika button dengan ID `cancel` diklik

### âš¡ Kenapa setTimeout(0) Tetap Berjalan Terakhir?

**Percobaan:**

**Output:**

> â“ **Mengapa demikian?**
> Meskipun timeout-nya 0 detik, callback tetap dijalankan **setelah** kode di global scope selesai.

**Urutan Eksekusi:**

```
2. setTimeout callback masuk ke Task Queue â³
```

> ğŸ’¡ **Kesimpulan:** Callback `setTimeout` selalu menunggu Call Stack kosong terlebih dahulu, bahkan jika delay-nya 0ms.
